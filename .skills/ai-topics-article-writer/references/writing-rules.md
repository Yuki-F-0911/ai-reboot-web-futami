# Writing Rules

## ターゲット読者

- **主要**: 生成AIを学び始めた社会人・ビジネスパーソン（IT専門知識なし）
- **副次**: 業務でAI活用を検討している管理職・経営者

## 文体ルール

### 基本
- 「です/ます」調で統一
- 1文は60字以内を目安
- 難解な英語略語は初出時に日本語を添える（例: LLM（大規模言語モデル））
- 業界用語はリンクで `/glossary/` に誘導し、本文では平易に言い換える

### トーン
- 中立・客観（特定製品の過度な礼賛は避ける）
- 「〜が注目されています」「〜が増えています」など柔らかめの語尾
- 緊張感を煽らない（「急がないと乗り遅れる」はNG）

## 禁止表現（AIスロップ）

以下の表現は使わない:
- 「革命的」「ゲームチェンジャー」「パラダイムシフト」
- 「まさに」「まさしく」「言うまでもなく」
- 「〜という時代に突入しました」
- 「〜が問われています」（乱用禁止）
- 「ぜひ〜」（CTA以外での使用）
- 「〜してみてはいかがでしょうか」（語尾として弱い）

## 必須要素

| 要素 | 仕様 |
|---|---|
| リード文 | 今週の全体像を2〜3文。「なぜ今週重要か」で締める |
| 各トピック | タイトル・解説（200〜400字）・ビジネスへの示唆1文 |
| 出典 | 各トピックに最低1つ（公式発表・信頼できるメディア） |
| 実践アクション | 3つ、具体的・今すぐできる行動 |
| 内部リンク | `/academy/blog/` 最低1本・`/glossary/` 最低1本 |
| LINE CTA | AiTopicArticle.tsx が自動挿入するため本文には書かない |

## 情報の扱いルール

- 変動しやすい情報（料金・機能・制度）は「確認日: YYYY-MM-DD」を付ける
- 不確実・未確認の情報は `[要確認: 理由]` としてラベリング
- X（Twitter）の投稿を引用する場合は匿名化し、個人特定情報を含めない
- Grok による分析は「AI分析」ではなく「各社の発表や報道によると」などソース重視の表現に変換

## SEO 最適化

- title: 「{YYYY}年{M}月第{N}週 生成AIニュース」等の週次キーワードを含む
- description: 120字以内、今週のメインテーマと対象読者を明示
- tags: 4〜6個、「週次まとめ」は必須、残りはトピック固有のキーワード

## 記事量の目安

| セクション | 文字数 |
|---|---|
| リード文 | 100〜150字 |
| 各トピック（3〜5本） | 200〜400字/本 |
| 実践アクション | 各1〜2文 |
| 次に読む | 2〜3リンク |
| **合計** | **1,500〜3,000字** |

## 内部リンク候補

これ以外のスラッグを使う場合は `ls data/glossary/` または `ls app/(site)/academy/blog/` で実在確認すること。

- `chatgpt-prompt-beginner` — ChatGPT プロンプト入門
- `ai-workplace-introduction-guide` — AI業務導入ガイド
- `claude-how-to-use` — Claude の使い方

- `agent` — AIエージェント
