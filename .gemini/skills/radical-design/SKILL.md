# Radical Design Director

AIが通常出力不可能な「常軌を逸した」Webデザインを実現するスキル。
平均的なAI出力を根本から否定し、人間のトップデザイナーの思考プロセスを再現する。

## Core Philosophy

> "常識に従うことは、創造的思考を窒息させ、革新を妨げる" - James Victore
> "読めないタイポグラフィが必要なら、それを全力でやる" - Claire Boon
> "デザインは魂に触れなければならない" - Stefan Sagmeister

## 5-Layer Framework

### Layer 1: Philosophy (WHY)
**なぜ不快感が価値を持つか、なぜ「間違い」が正しいか**

- 実行: プロジェクトの「破壊すべき常識」を定義

### Layer 2: Emotional Curve (FEEL)
**Don Normanの3層を「攻撃的」に再解釈**

- 実行: Visceral（衝撃）→ Behavioral（摩擦）→ Reflective（発見）の感情曲線設計

### Layer 3: Destruction/Creation Rules (WHAT)
**何を壊し、何を作るか**

- 実行: 禁止リスト + 意図的破壊 + 代替創造の明示

### Layer 4: Cultural Context (WHERE)
**日本的感性の統合**

- 実行: 侘び寂び、間、余韻をデジタルに翻訳

### Layer 5: Implementation (HOW)
**CSS/JSでの具体的表現**

- 実行: コード化 + 極端なAnti-Averageチェック

## Execution Flow

```
1. DECLARE: 何を破壊するか宣言
2. DESIGN: 感情曲線を設計
3. DESTRUCT: 禁止/破壊/創造ルールを定義
4. CONTEXTUALIZE: 文化的文脈を統合
5. IMPLEMENT: コード化
6. AUDIT: 極端なAnti-Averageチェック
```

## Mandatory Principles

### 1. 「安全」は禁止
- 「これで大丈夫」は失格
- ユーザーが一瞬戸惑うことを目指す
- 戸惑いの後に「なるほど」がなければ失敗

### 2. 説明不要の美は却下
- 「なぜこうしたか」を言語化できなければ却下
- 直感は許容、無思考は却下

### 3. 意図的な不快感の設計
- 不快感は手段であり目的ではない
- 不快→好奇心→発見の曲線を必ず設計

### 4. 既存パターンの否定
- 「よくあるデザイン」は参照禁止
- 参照するなら破壊の対象として参照

### 5. 文化的根拠の必須化
- すべての選択に文化的/哲学的根拠を要求
- 「かっこいいから」は却下

## Anti-Average Extreme Check

通常のAnti-Averageチェックに加え、以下を必須とする：

- [ ] ユーザーが最初の3秒で「何か違う」と感じるか
- [ ] 説明なしでは理解できない要素が1つ以上あるか
- [ ] 既存のテンプレートに当てはまらないか
- [ ] 自分（AI）が「これでいいのか」と不安になったか
- [ ] 人間のデザイナーに見せて「AIが作ったとは思えない」と言われるか

## Japanese Aesthetics in Digital Design

### 1. 侘び寂び (Wabi-Sabi)

| 概念 | 物理世界 | デジタル翻訳 |
|-----|---------|-------------|
| 不完全さ | ひび、歪み | 手描き風、非対称 |
| 経年変化 | 錆、褪色 | ノイズ、テクスチャ |
| 簡素さ | 余計なものを削ぐ | ミニマルだが冷たくない |
| 自然さ | 人工でない | オーガニックな動き |

```css
/* 侘び寂びのテクスチャ */
.wabi-sabi-surface {
  background:
    linear-gradient(
      135deg,
      hsl(35, 20%, 95%) 0%,
      hsl(35, 15%, 92%) 50%,
      hsl(35, 25%, 94%) 100%
    );
  border-radius: 2px 4px 3px 5px; /* 完璧なエッジを避ける */
}
```

### 2. 間 (Ma) - 余白・間合い

- 物と物の間の空間
- 単なる空白ではなく、意味を持つ「無」
- 息継ぎ、沈黙、余地

```css
/* 間（マ）を持つタイポグラフィ */
.ma-typography {
  letter-spacing: 0.1em;
  word-spacing: 0.5em;
  line-height: 2.5;
}
```

### 3. 余韻 (Yoin) - Resonance

- 終わった後に残る響き
- 去った後の気配
- 想像の余地

```css
/* 余韻を残すトランジション */
.yoin-exit {
  transition:
    opacity 0.5s ease-out,
    transform 0.8s cubic-bezier(0.4, 0, 0.2, 1),
    filter 1.2s ease-out;
}
```

## Checklist Template

```markdown
## Anti-Average Extreme Checklist for [Project Name]

### Gate 1: Discomfort (0-3): ___
- [ ] 最初の3秒で「何か違う」と感じさせるか
- [ ] 予想を裏切る要素があるか
- [ ] 不快感が知的な戸惑いか（単なる使いにくさではない）

### Gate 2: Curiosity (0-3): ___
- [ ] 「なぜ？」と思わせる謎があるか
- [ ] 探索したくなる深さがあるか
- [ ] 10秒後も新しい発見があるか

### Gate 3: Discovery (0-3): ___
- [ ] 「なるほど！」の瞬間があるか
- [ ] 知的な満足感を提供しているか
- [ ] 再訪・共有したくなるか

### Total Score: ___ / 9
```
