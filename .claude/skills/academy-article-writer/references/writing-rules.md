# 文体ルール

## 基本方針
「羅列・網羅」より「判断基準・Before/After・実務文脈」を優先する。
各H2は「結論→現状→解決策→実行手順」の順で構成。

## AIスロップ禁止表現

### 比喩・メタファー（使わない）
羅針盤、地図、航路、コンパス、灯台、道しるべ、武器、盾、旅・旅路

### 煽り・大げさ表現（使わない）
地獄、殴る、刺さる、勝つ・負ける・戦う、革命、爆発的、劇的、圧倒的（数値なし）、〜の沼、〜の罠

### AI定型フレーズ（使わない）
- 「〜の世界へようこそ」「さあ、始めましょう」「さっそく見ていきましょう」
- 「この記事を読めば〜がわかります」
- 「〜の重要性は言うまでもありません」
- 「ご存知でしょうか」「実は〜なのです」
- 「〜に他なりません」「〜に違いありません」

### 代わりに
比喩を使わず具体的な動詞・数値・実務事例で書く。

## E-E-A-T
- 経験: 「〜の現場では」「〜の事例では」
- 専門性: 用語を正確に使い、判断根拠を示す
- 権威性: 公的指針への参照（「デジタル庁のガイドライン」等、URL直書き不要）
- 信頼性: 確認できない数値は `[要確認: XX%]` で明示

## 一次情報・実利用者の声の扱い
- 断定は一次情報を優先し、SNSの声は「補助根拠」として扱う
- SNS投稿は個人情報/固有IDを出さず、要点を要約して紹介する
- 価格・仕様・制度など変動しやすい情報は確認日を添える

## LINE CTA 設計方針

**ポジショニング**: AIリブートのLINEは「週1本のAI学習コンテンツ＋ニュース解説を届ける無料予習チャンネル」。
記事の種類（ツール系・基礎系・キャリア系）にかかわらず、CTA は以下の1パターンに統一する。

**標準CTA文言（全記事共通）**:
```
AIリブート通信｜週1本、仕事で使えるAI知識＋ニュース解説をLINEで届ける（無料）
```
ボタン短縮テキスト: 「LINEで週1AI通信を受け取る（無料）」

**NG パターン（使わない）**:
- 「〇〇スターターテンプレをLINEで受け取る」（ツール固有素材が必要）
- 「〇〇チェックリストをダウンロード」（記事ごとに素材が必要）
- 「LINE登録する」（何が得られるか不明）
- 「ダウンロードはこちら」（同上）

**配置**: 記事内に2〜3箇所（H2-1直後・本文中盤・FAQ末尾）

## アカデミーCTA ポジショニング（CRITICAL）

記事末尾のアカデミー誘導CTAは、**アカデミーが実際に提供する価値の3本柱**を軸に書くこと。
記事のテーマ（特定ツール・技術）をそのままアカデミーで学べる場所として繋ぐのは**厳禁**。

### アカデミーが提供する3本柱

| 柱 | 内容 |
|---|---|
| **生成AI活用力** | 実務で即使えるAIスキルを体系的に習得 |
| **自己理解・キャリアデザイン** | AIを鏡に自分の強み・価値観を掘り下げ、キャリアを再設計する |
| **仲間と共に学ぶ環境** | 志を同じくする仲間との対話・協働が変化を加速させる |

### アカデミーが提供しないもの

- 特定ツールの操作手順・細かい使い方（Dify の設定方法、Power Automate の操作、RAG の実装コードなど）
- プログラミング言語の習得（Python等）
- AIエージェント・ノーコードツールの構築スキル
- LLMの評価・運用の技術実装

### NG パターン（使ってはいけない）

```
❌ 「DifyをはじめとするAIエージェント・ノーコードAIツールの活用方法を体系的に学べるプログラム」
   → 特定ツールの使い方を教える場所と誤解させる

❌ 「AIツール操作の習得だけでなく」「ツールの操作方法だけでなく」
   → 前提として「ツール操作を習得できる」という印象を与える

❌ 「導入判断を実装できる力に変える」
   → 技術実装スキルを教えるかのよう

❌ 「Python×AIを実務で使える型へ」「RAGを成果が出る運用へ」「AIエージェントを実務で回る仕組みへ」
   → 記事テーマの技術スキルをそのまま習得できると誤解させる

❌ 「〜の活用方法を体系的に学べる」「〜を学べるプログラム」（特定ツール名を主語にした場合）
   → アカデミーがそのツール専門スクールに見える
```

### OK パターン（使うべき表現）

```
✅ 「AIリブートアカデミーでは、生成AI活用力の習得だけでなく、
     AIを鏡にした自己理解・キャリアデザイン、仲間と共に学ぶ環境づくりまで一体で設計しています。
     〇〇のようなツールをどの業務課題に当てるか、活用の判断軸と実務への組み込み方を
     考えたい方は、学習プロセス全体を見直すことが有効です。」

✅ 「ツール名より『何を伝えて、相手にどう動いてほしいか』を言語化する思考OSが成果を決めます。
     AIリブートアカデミーでは、3本柱（生成AI活用力・自己理解/キャリアデザイン・仲間と学ぶ環境）で
     実務アウトプットとキャリアの次の一歩へ落とし込みます。」
```

### CTA タイトルの作り方

記事テーマをそのままタイトルにせず、3本柱の文脈で締めること。

| 記事テーマ | NG タイトル | OK タイトル |
|---|---|---|
| Dify / ノーコードAI | 「Difyの活用方法を体系的に学ぶ」 | 「AI活用の判断軸とキャリアを同時に設計する」 |
| RAG / ファインチューニング | 「実装できる力に変える」 | 「AIの判断軸をキャリアの武器に変える」 |
| エージェント構築 | 「エージェントを実務で回る仕組みへ」 | 「AI活用力とキャリアを同時に設計する」 |
| Python × AI | 「Python×AIを実務で使える型へ」 | 「AIを活用する判断力とキャリアの武器を育てる」 |
