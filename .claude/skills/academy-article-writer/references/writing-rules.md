# 文体ルール

## 基本方針
「羅列・網羅」より「判断基準・Before/After・実務文脈」を優先する。
各H2は「結論→現状→解決策→実行手順」の順で構成。

## AIスロップ禁止表現

### 比喩・メタファー（使わない）
羅針盤、地図、航路、コンパス、灯台、道しるべ、武器、盾、旅・旅路

### 煽り・大げさ表現（使わない）
地獄、殴る、刺さる、勝つ・負ける・戦う、革命、爆発的、劇的、圧倒的（数値なし）、〜の沼、〜の罠

### AI定型フレーズ（使わない）
- 「〜の世界へようこそ」「さあ、始めましょう」「さっそく見ていきましょう」
- 「この記事を読めば〜がわかります」
- 「〜の重要性は言うまでもありません」
- 「ご存知でしょうか」「実は〜なのです」
- 「〜に他なりません」「〜に違いありません」

### 代わりに
比喩を使わず具体的な動詞・数値・実務事例で書く。

## E-E-A-T
- 経験: 「〜の現場では」「〜の事例では」
- 専門性: 用語を正確に使い、判断根拠を示す
- 権威性: 公的指針への参照（「デジタル庁のガイドライン」等、URL直書き不要）
- 信頼性: 確認できない数値は `[要確認: XX%]` で明示

## LINE CTA 設計方針

**ポジショニング**: AIリブートのLINEは「週1本のAI学習コンテンツ＋ニュース解説を届ける無料予習チャンネル」。
記事の種類（ツール系・基礎系・キャリア系）にかかわらず、CTA は以下の1パターンに統一する。

**標準CTA文言（全記事共通）**:
```
AIリブート通信｜週1本、仕事で使えるAI知識＋ニュース解説をLINEで届ける（無料）
```
ボタン短縮テキスト: 「LINEで週1AI通信を受け取る（無料）」

**NG パターン（使わない）**:
- 「〇〇スターターテンプレをLINEで受け取る」（ツール固有素材が必要）
- 「〇〇チェックリストをダウンロード」（記事ごとに素材が必要）
- 「LINE登録する」（何が得られるか不明）
- 「ダウンロードはこちら」（同上）

**配置**: 記事内に2〜3箇所（H2-1直後・本文中盤・FAQ末尾）
