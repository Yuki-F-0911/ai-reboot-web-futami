# デザイン・実装仕様

## 1. 全体のデザイントーン

### 目指す印象
- 挑戦感がある
- 実践感がある
- 熱量がある
- ただの学校行事っぽくない
- 外部の人が見ても価値が分かる

### カラーパレット（推奨）
- メインカラー: #1a1a2e（ダークネイビー）── 信頼感・プロフェッショナル
- アクセントカラー: #e94560（レッド系）── 挑戦・熱量・CTA
- サブカラー: #0f3460（ディープブルー）── 見出し・セクション背景
- テキスト: #f5f5f5（ライトグレー on ダーク）/ #1a1a2e（ダーク on ライト）
- 背景: グラデーション使用可（ダーク系ベース推奨）
- DAY 100 ブロック: ゴールド系（#ffd700）のアクセントで特別感を演出

### フォント
- 見出し: Noto Sans JP Bold / 700
- 本文: Noto Sans JP Regular / 400
- Google Fonts から読み込み

## 2. セクション別デザイン指示

### ① ファーストビュー
- フルスクリーン表示（100vh）
- ダーク背景 + 大きなキャッチコピー（白文字）
- 開催日時・形式を目立つ位置に配置
- CTA ボタンを画面下部に配置（アクセントカラー、角丸、ホバーアニメーション）
- 背景にはアブストラクトなグラデーションやパーティクル風の軽い装飾を CSS で

### ② DEMO DAYとは
- ライト背景に切り替え
- 中央寄せのテキストブロック
- シンプルで読みやすく

### ③ 当日の見どころ
- ダーク背景に戻す（メリハリ）
- 見出しを大きく
- 5つのフィールド（建築、ゲーム開発…）をアイコン付きで横並び表示（モバイルは縦）

### ④ 5名が向き合ってきたテーマ（タイムライン）★最重要
- 各受講生ごとにカード形式のブロック
- カード内にタイムライン（DAY 1 → DAY 50 → DAY 100 を縦に3ステップ）
- DAY 1, DAY 50: 通常表示。引用符（" "）で受講生コメントを表示
- DAY 100: 特別デザイン
  - 背景をゴールド/アクセントカラーのグラデーションに
  - 鍵アイコン（CSSまたは絵文字で）
  - 「??? ── DEMO DAY 当日」のテキスト
  - ティーザーテキストを半透明で表示
  - パルスアニメーションや光るボーダーで注目を引く
- アバターは絵文字を大きく表示（50px程度）
- ニックネームとフィールド（建築 × AI 等）をカード上部に

### ⑤ 発表で見られること
- ライト背景
- テキスト中心。読みやすさ重視

### ⑥ 当日のプログラム
- シンプルなテーブルまたはタイムライン表示
- 時間と内容を対で表示

### ⑦ 開催概要
- 定義リスト形式（dl/dt/dd）で整理して表示
- CTA ボタンをセクション末尾に配置

### ⑧ AIリブートアカデミーとは
- ライト背景
- 本文 + 3ステップの横並びカード（STEP 01/02/03）
- STEP 03 に「← 今回はここ！」のバッジ/ハイライトを付ける

### ⑨ FAQ
- アコーディオン形式（クリックで開閉）
- JavaScript で開閉トグルを実装

### ⑩ CTA再掲（フッター）
- ダーク背景
- 見出し + 日時情報 + CTA ボタン
- ボタンは大きめ、目立つアクセントカラー
- フッターにクレジット

## 3. 実装要件

### 必須
- スマホ最適化（モバイルファースト）
- CTA の複数配置（3箇所）
- 後から発表テーマを差し替えやすい構造（content.json のデータ構造に合わせてHTML内にコメントで目印を入れる）
- 匿名運用に対応しやすいレイアウト（顔写真なしでも成立するデザイン）

### 望ましい
- 5名分の情報更新がしやすい
- テーマ詳細を直前反映しやすい
- 写真なしでも成立するデザインテンプレート
- スクロールアニメーション（Intersection Observer で要素が画面に入ったらフェードイン）
- スムーズスクロール（CTA クリック時）

## 4. 注意点
- 当日の発表価値を先に消費しないようにする（DAY 100 の内容は必ずブラックボックス）
- 抽象的すぎても弱い。「見どころの輪郭」は必ず伝わるようにする
- テーマ詳細未確定前提の設計が必要（テーマ確定前でも成立する構成にする）
- リアル来場情報は混ぜない（LP上はオンライン参加のみで統一）
- 個人情報：本名は原則非公開、顔写真も原則非公開。ニックネーム + フィールド + コメントで構成

## 5. ファイル構成

プロジェクトルートに以下のファイルを配置:

```
/
├── index.html      ← メインファイル（HTML + CSS + JS 全て含む）
├── PROMPT.md        ← この制作指示書（参考用）
├── content.json     ← コンテンツデータ（参考用。HTML 内にハードコード）
└── design-spec.md   ← このファイル（参考用）
```

※ index.html は1ファイル完結。content.json は参考データであり、HTML 内に直接テキストを埋め込む。
