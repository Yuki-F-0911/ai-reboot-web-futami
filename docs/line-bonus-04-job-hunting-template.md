## 6. 面接想定Q&A生成プロンプト（5問）

> 使い方：求人票と職務経歴書を貼り、1本ずつ実行してください。  
> 出力形式はすべて「質問→回答案（結論→根拠→再現性）」です。

### プロンプト1：自己紹介（1分）を作る

```text
あなたは採用面接官です。以下の情報を使って、一次面接で聞かれる「自己紹介してください」に対する回答案を作成してください。

[入力]
- 応募職種: {応募職種}
- 職務経歴書要約: {職務経歴書要約}
- 強み: {強み3つ}

[出力条件]
- 300文字以内
- 結論→実績→応募先での貢献の順
- 曖昧語（頑張りました、努力しました）を使わない
```

### プロンプト2：転職理由を一貫化する

```text
以下をもとに「転職理由」を面接向けに作成してください。

[入力]
- 現職の課題: {現職の課題}
- 次にやりたいこと: {次にやりたいこと}
- 応募先を選ぶ理由: {応募先理由}

[出力条件]
- 200〜250文字
- ネガティブ退職理由だけで終わらない
- 現職批判を避ける
- 最後は応募先での価値提供で締める
```

### プロンプト3：実績深掘り質問に備える

```text
あなたは面接官です。以下の実績に対して、深掘り質問を5問作成し、それぞれに模範回答を作ってください。

[入力]
- 実績: {実績本文}
- 役割: {役割}
- 制約条件: {制約}

[出力形式]
1) 質問
2) 回答案（結論→行動→結果→学び）
```

### プロンプト4：失敗経験の回答を作る

```text
以下の失敗事例を使って、面接で使える回答を作成してください。

[入力]
- 失敗事例: {失敗事例}
- 原因: {原因}
- 再発防止策: {再発防止策}
- その後の成果: {改善後成果}

[出力条件]
- 250〜350文字
- 責任転嫁しない
- 学びと再現可能な改善行動を明確にする
```

### プロンプト5：逆質問を5問作る

```text
あなたは転職候補者の面接コーチです。以下の条件で、評価されやすい逆質問を5問作成してください。

[入力]
- 応募企業の事業内容: {事業内容}
- 募集職種: {募集職種}
- 気になる論点: {成長機会/評価制度/チーム体制/課題など}

[出力条件]
- 調べれば分かる質問を避ける
- 入社後の貢献イメージが伝わる質問にする
- 各質問に「意図（なぜ聞くか）」を1行添える
```

## 7. 提出前チェックリスト（必須）

- [ ] 役職名・期間・数値に誤りがない  
- [ ] 成果は「行動」とセットで記述した  
- [ ] 応募先ごとにキーワード（求人票の表現）を反映した  
- [ ] AI生成文をそのまま使わず、本人の言葉に整えた  
- [ ] 面接想定Q&Aを最低1回音読した
