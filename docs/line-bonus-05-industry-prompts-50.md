## 2. 使い方（最短3ステップ）

1. 変数 `{}` を自分の業務情報に置換する  
2. 出力形式を必ず指定して実行する  
3. 事実確認してから提出・送信する

---

## 3. 共通変数（置換して使う）

- `{目的}`：今回達成したいこと
- `{前提}`：相手・状況・制約
- `{入力データ}`：元情報（議事録・数値・メモ等）
- `{出力形式}`：箇条書き、表、メール文、要約など
- `{文字数}`：200字、500字など

## 4. 営業向けプロンプト（10選）

### 営業01：初回ヒアリング質問設計
```text
あなたは法人営業のアシスタントです。以下の商談に向けて、初回ヒアリング質問を10個作成してください。
商材: {商材}
相手業種: {業種}
目的: {目的}
出力: 「質問」「確認意図」「次に掘る質問」の3列テーブル
```

### 営業02：提案メール初稿
```text
あなたは営業担当です。以下の条件で提案メール初稿を作成してください。
宛先: {宛先}
提案内容: {提案内容}
締切: {締切}
出力: 件名→本文（300字以内、丁寧語、次アクション明記）
```

### 営業03：失注分析
```text
次の失注メモを分析し、再提案の改善点を整理してください。
入力: {失注メモ}
出力: 1)失注要因 2)改善仮説 3)次回商談アクション（3件）
```

### 営業04：提案書アウトライン生成
```text
以下をもとに提案書の目次を作成してください。
顧客課題: {課題}
提案骨子: {骨子}
競合状況: {競合}
出力: 8章構成、各章に「目的」と「入れるべき根拠」を記載
```

### 営業05：商談議事録の要約
```text
以下の議事録を、営業フォロー用に要約してください。
議事録: {議事録}
出力: 1)決定事項 2)未決事項 3)次回アクション 4)担当者 5)期限
```

### 営業06：見積説明トーク作成
```text
以下の見積内容を顧客向けに説明するトークスクリプトを作成してください。
見積内訳: {見積内訳}
懸念点: {懸念点}
出力: 90秒で話せる説明文。価格根拠と費用対効果を含める
```

### 営業07：反論対応Q&A
```text
以下の商材に対する想定反論と回答を10セット作成してください。
商材: {商材}
顧客属性: {顧客属性}
出力: 「想定反論」「回答」「補足資料」の3列
```

### 営業08：日報自動生成
```text
次の活動ログから営業日報を作成してください。
活動ログ: {活動ログ}
出力: 1)実施内容 2)成果 3)課題 4)翌日計画（計400字以内）
```

### 営業09：優先顧客の選定
```text
次の顧客リストから優先度を判定してください。
顧客データ: {顧客データ}
判定基準: 受注確度、案件規模、意思決定スピード
出力: A/B/Cランクと理由、次アクション
```

### 営業10：月次レビュー資料の骨子
```text
以下のKPIを使って、月次レビュー資料の骨子を作成してください。
KPI: {KPI}
前月比較: {前月比較}
出力: スライド8枚分の構成（スライド名、要点、必要グラフ）
```
