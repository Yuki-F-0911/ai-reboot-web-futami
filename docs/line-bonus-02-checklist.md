# AIリブートアカデミー｜AI導入効果チェックリスト（2〜3枚）

> 対象：AI活用を始めたばかりの個人・チーム向け  
> ゴール：**「やってる感」ではなく、数字と習慣で“定着”させる**

---

## 0. 免責事項（必須）

本資料は一般的な情報提供を目的としたテンプレートであり、法的助言・個別具体のコンサルティングではありません。最終判断は所属組織の規程および関係部門（法務/セキュリティ等）とご確認ください。

## 1. 導入前の業務棚卸しチェックリスト（20項目）

> まずは「AIで何でも」ではなく **“週に何回ある反復タスク”** を洗い出します。

- [ ] 週に3回以上発生する「定型文章作成」がある（メール、案内文、返信テンプレなど）
- [ ] 議事録/メモ/録音の要約が負担になっている
- [ ] 情報収集→要約→比較（資料読み）が多い
- [ ] 仕様/要件/タスクの“整理”に時間がかかる
- [ ] FAQ対応（同じ質問に何度も答える）がある
- [ ] Excel/スプレッドシートの整形・集計・関数が手作業になっている
- [ ] 日次/週次レポートの作成がルーチン化している
- [ ] 提案書/企画書の「構成づくり」で止まりやすい
- [ ] 画像/バナー/サムネなどの制作がボトルネックになっている
- [ ] SNS/ブログなど短文の量産が必要
- [ ] 翻訳（JP↔EN）や言い換えが頻繁にある
- [ ] 会議のアジェンダ作成が属人化している
- [ ] 問い合わせ対応の一次切り分けができていない
- [ ] 社内ナレッジが散在して探すのに時間がかかる
- [ ] コード/SQL/スクリプトを「書く or 読む」作業がある（非エンジニア含む）
- [ ] バグ/エラーの切り分けが遅い（再現・ログ整理が苦手）
- [ ] 依頼/タスクの粒度が粗く、手戻りが多い
- [ ] 仕組み化（自動化/ワークフロー）したいが着手できていない
- [ ] 品質チェック（誤字脱字、体裁、観点漏れ）が最後に集中する
- [ ] 「AIを使っていい/ダメ」の判断基準が曖昧で止まることがある

## 2. AI適性判定マトリクス（タスク×難易度×効果）

> “効くタスク”は **(A)反復性** と **(B)入力が整っている** の2つで決まります。

### 2-1. 判定ルール（3つだけ）

- **難易度（低→高）**：入力が揃っている／判断が必要／例外が多い
- **効果（低→高）**：発生頻度が高い／時間が長い／品質要求が高い（ミスが痛い）
- **推奨**：まずは「低難易度×高効果」から着手（小さく勝つ）

### 2-2. マトリクス表（記入テンプレ）

| タスク | 難易度（低/中/高） | 効果（低/中/高） | リスク（低/中/高） | 推奨アプローチ（例） |
|---|---|---|---|---|
| `[TODO] 議事録要約` | 低 | 高 | 中 | テンプレ+確認観点（固有名詞/数値） |
| `[TODO] FAQ一次回答` | 中 | 高 | 中 | ナレッジ整備→下書き生成→人が最終回答 |
| `[TODO] 提案書構成` | 中 | 中 | 中 | アウトライン生成→社内事例で肉付け |
| `[TODO] 顧客向け文面` | 中 | 高 | 高 | 下書きのみ、承認フロー必須 |
| `[TODO] 自動化（送信/更新）` | 高 | 高 | 高 | 権限/ログ/ロールバック設計が先 |

### 2-3. 豆知識：失敗しやすいパターン（予防）

- 難易度が高いのに、いきなり自動化（例外処理で破綻）
- 効果が低いのに、プロンプトに時間を溶かす（“AIのための仕事”化）
- 出力をそのまま提出（誤情報・トーン崩れ・機密混入）
