## 2. データ解釈への指摘（問題点と改善提案）
### 2-1. 「+37.8% セッション増」は本当に喜ぶべきか
- 問題:
  - 比較期間が1日ズレており、純粋なWoWではない（6日重複比較）。
  - 前期494は暫定値、今期681は成熟後値で、比較条件が非対称。
  - 4週参照平均（利用可能データ: `84, 494, 681, 13`）は `318`。681は平均超えだが、`13` はT+24h暫定で歪みが大きい。
- 改善提案:
  - 次回から「非重複7日比較」を固定（例: Mon-Sunで統一）。
  - `T+24h暫定` と `T+72h確定` を明示分離し、確定値のみでトレンド評価。
  - 移動平均は `平均` だけでなく `中央値` も併記（現4点の中央値は `289`）。

### 2-2. CVが同値（briefing2/contact2/line1）の意味
- 事実:
  - CV総数は `5→5`（増加ゼロ）。
  - セッション増に対して効率は低下: 総CVR `1.01%→0.73%`、line CVR `0.20%→0.15%`。
- 解釈:
  - 施策効果がゼロとは断定できないが、少なくとも「トラフィック増をCVに転換できていない」ことは確定。
  - Direct 102セッション（直帰89.2%）が分母を膨らませている可能性が高い。仮にDirect除外で再計算すると総CVRは `5/579=0.86%` まで戻るが、それでも前期 `1.01%` 未満。
- 改善提案:
  - CVRを `全体` と `Organic-only` の2軸で常時表示。
  - CTA施策は「実装数」ではなく「表示回数→クリック率→登録率」の漏斗で評価。

### 2-3. CTR 1.17%（-0.17pt）の意味
- 事実:
  - クリック `209→263`（+25.8%）だが、インプレッション `15,609→22,572`（+44.6%）の伸びに負けてCTR低下。
  - 旧CTR1.34%を維持できていれば想定クリックは約 `303`（22,572×1.34%）で、実績263との差は約 `40クリック`。
  - 平均順位は `9.9→10.3`（-0.4）とわずかに悪化。
- 解釈:
  - 原因は「順位低下だけ」ではない。順位が悪くないのに低CTRなページが目立つ。
  - 例: `ai-trends-february-2026`（`1,236imp / CTR0.5% / 順位4.6`）、`chatgpt-plus-honest-review-2026`（`865imp / CTR0.6% / 順位6.7`）。
- 改善提案:
  - CTR改善は title/description最適化を最優先で継続（現行方針は妥当）。
  - ただし「順位帯別CTR（1-3位/4-10位/11位以下）」を次回必須出力にして、要因を分解する。

## 4. 見落とされている重要な視点
### 4-1. CTR低下の根因分解不足
- 現状は「低CTRだからtitle改善」で止まっている。
- 追加で必要:
  - `平均順位変化` と `CTR変化` の相関確認（ページ別）。
  - クエリ意図ズレ（情報収集系 vs 比較/購入意図）ごとのCTR分解。
  - title変更前後で `同一ページ・同一順位帯` の差分追跡。

### 4-2. line_register_click（3週連続0〜1件）の根本仮説
- 3週推移: `1 → 1 → 0`。KPI目標 `3/週` と乖離。
- 仮説:
  - 露出不足: 25%到達190に対し75%到達74（39%）。記事末CTA中心だと母数が不足。
  - 文脈不一致: 記事テーマ別のCTA文言最適化が弱い（デフォルト依存）。
  - 計測漏れ: クリックはあるが `line_register_click` が落ちていない可能性を未検証。
- 追加検証:
  - `page_path` 別の line_register_click 集計を必須化。
  - `CTA表示回数` を計測して、CTR（クリック/表示）で評価。

### 4-3. A-3（workflow-automation-comparison CTA追加）の効果測定不足
- 実装はDoneだが、今週実績は `1PV / 203s / CV0`。サンプル不足で結論不可。
- 3/9暫定版でも同ページ `1PV / CV0` のまま。改善/悪化どちらも判定不能。
- 施策評価条件を「最低セッション母数（例: 30）」で事前定義すべき。

  - `13` が確定で大幅上振れするなら、暫定取得タイミングの問題。
