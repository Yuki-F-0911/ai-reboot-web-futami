- 管理職、営業、マーケ、経営企画、バックオフィス

- Xっぽい温度感は使うが、投稿引用はしない

## H2構成

### H2-1: GPT-5.4がビジネスを変える3つの理由
- 結論: 変化の本質は `精度` `長文処理` `ツール横断`
- H3:
  - エラーが減り、レビュー回数が減る
  - 1Mコンテキストで数百ページをまとめて扱える
  - PC操作、Web調査、スプレッドシートまで一気通貫になった

### H2-2: 職種別活用10選
- 顧客対応: サポートチケットの自動化
- マーケ: 広告キャンペーン一式の初稿
- 管理職/PM: 会議録・長文資料の5行要約
- 財務: Excelモデリング
- 法務/調達: 契約書・仕様書の比較
- 営業: メール下書き・報告書の敬語調整
- 海外営業/人事: 英訳・ローカライズ
- 企画: アイデア出し・企画書骨格
- 経営企画/マーケ: 競合調査・市場リサーチ
- 非エンジニア部門: プログラミング不要のデータ分析

### H2-3: 今週から使える具体的な手順
- 1業務だけ選ぶ
- Before の所要時間を計る
- プロンプトを `役割 / 入力 / 出力形式 / 禁止事項` で固定
- レビュー観点を決める
- 週次で KPI を見て残す

### H2-4: コスト試算
- Free / Plus / Pro / API の向き不向き
- API の月額試算:
  - 軽運用: 2M input + 0.3M output = $9.5
  - 中運用: 10M input + 1M output = $40
- コストより `何時間戻るか` で判断

### H2-5: 注意点・限界
- 機密情報はそのまま入れない
- 数字・日付・契約文言は人が最後に確認
- 1M context は API/Codex 側の話で、ChatGPT Web と同一ではない
- 自動化率はナレッジ品質とエスカレーション設計に依存

### H2-6: FAQ
- GPT-5.4は誰向けか
- 無料/Plus/Proのどれで始めるべきか
- Excelはどこまで任せられるか
- 契約書や仕様書は安全に扱えるか
- サポート自動化は本当に70%までいけるか
- どのKPIから見ればいいか
