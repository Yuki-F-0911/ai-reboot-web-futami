# MicroCMS スキーマ設定ガイド

## 概要
このドキュメントは、AI REBOOTサイトで使用するMicroCMSのスキーマ設定について説明します。

## 1. APIエンドポイント設定

### news API（リスト形式）
お知らせとブログ記事の両方を管理するAPIです。

## 2. フィールド設定

### 基本フィールド

| フィールドID | 表示名 | フィールド形式 | 必須 | 説明 |
|-------------|--------|--------------|------|------|
| title | タイトル | テキストフィールド | ○ | 記事のタイトル |
| category | カテゴリー | セレクトフィールド | ○ | 記事のカテゴリー |
| description | 概要 | テキストエリア | - | 記事の概要（SEO用） |
| content | 本文 | リッチエディタ | ○ | HTML形式の本文 |
| md-content | マークダウン本文 | テキストエリア | - | Markdown形式の本文 |
| thumbnail | サムネイル画像 | 画像 | - | 記事のサムネイル |
| tags | タグ | 複数テキストフィールド | - | 記事のタグ（SEO用） |

### カテゴリー設定（セレクトフィールド）

#### お知らせカテゴリー
- `news` - ニュース
- `event` - イベント
- `media` - メディア掲載
- `notice` - お知らせ

#### ブログカテゴリー
- `featured` - 注目記事
- `ai-trends` - AIトレンド
- `case-study` - 活用事例
- `tutorial` - チュートリアル
- `prompts` - プロンプト集
- `tools` - ツール紹介

## 3. フィールドの詳細設定

### title（タイトル）
- **フィールドID**: `title`
- **フィールド形式**: テキストフィールド
- **必須**: はい
- **文字数制限**: 100文字以内を推奨
- **説明**: SEOを考慮した、わかりやすいタイトルを入力

### category（カテゴリー）
- **フィールドID**: `category`
- **フィールド形式**: セレクトフィールド
- **必須**: はい
- **選択肢**（日本語で設定）:
  
  #### ブログカテゴリー
  - 注目記事
  - AIトレンド
  - 活用事例
  - チュートリアル
  - プロンプト集
  - ツール紹介
  
  #### お知らせカテゴリー
  - ニュース
  - イベント
  - メディア掲載
  - お知らせ
  
  **注意**: MicroCMSのセレクトフィールドは表示名しか設定できないため、日本語で設定します。システム側で自動的に英語IDにマッピングされます。

### description（概要）
- **フィールドID**: `description`
- **フィールド形式**: テキストエリア
- **必須**: いいえ
- **文字数制限**: 160文字以内を推奨
- **説明**: 検索結果やSNSシェア時に表示される概要文

### content（HTML本文）
- **フィールドID**: `content`
- **フィールド形式**: リッチエディタ
- **必須**: はい（md-contentがない場合）
- **説明**: HTMLエディタで記事を作成する場合に使用

### md-content（マークダウン本文）
- **フィールドID**: `md-content`
- **フィールド形式**: テキストエリア
- **必須**: いいえ
- **説明**: Markdown形式で記事を作成する場合に使用
- **注意**: ハイフンを含むフィールドIDのため、コード内では `article['md-content']` でアクセス

### thumbnail（サムネイル画像）
- **フィールドID**: `thumbnail`
- **フィールド形式**: 画像
- **必須**: いいえ
- **推奨サイズ**: 1200x630px（OGP画像として使用）
- **ファイル形式**: JPG, PNG, WebP

### tags（タグ）
- **フィールドID**: `tags`
- **フィールド形式**: 複数テキストフィールド
- **必須**: いいえ
- **説明**: SEO向上のためのタグ（複数設定可能）

## 5. SEO最適化のポイント

### タイトルの付け方
- キーワードを前方に配置
- 32文字以内が理想的
- 数字を含めると効果的（例：「5つの方法」「2024年版」）

### descriptionの書き方
- 120-160文字以内
- キーワードを自然に含める
- クリックしたくなる文言を使用

### タグの選定
- 関連性の高いキーワードを3-5個
- 一般的すぎず、具体的すぎないバランス
- 例：`ChatGPT`, `プロンプトエンジニアリング`, `業務効率化`

## 6. 画像の最適化

### サムネイル画像
- サイズ：1200x630px（OGP標準）
- ファイルサイズ：500KB以下
- テキストを含める場合は読みやすさを確保

### 本文内の画像
- 適切なalt属性を設定
- WebP形式を推奨（軽量）
- レスポンシブ対応を考慮

## 7. 自動振り分けの仕組み

### カテゴリーによる自動振り分け

MicroCMSで記事を作成する際、選択したカテゴリーによって自動的に表示場所が決まります：

#### ブログカテゴリーを選択した場合
以下のカテゴリーを選択すると `/blog` セクションに表示されます：
- 注目記事
- AIトレンド
- 活用事例
- チュートリアル
- プロンプト集
- ツール紹介

#### お知らせカテゴリーを選択した場合
以下のカテゴリーを選択すると `/news` セクションに表示されます：
- ニュース
- イベント
- メディア掲載
- お知らせ

### 重要な注意点
- **カテゴリーは必ず選択してください**：カテゴリーが未選択の記事は表示されません
- **正しいカテゴリーを選択**：ブログ記事をお知らせカテゴリーで作成すると、お知らせページに表示されてしまいます
- **URLの違い**：
  - ブログ記事: `https://ai-reboot.io/blog/[記事ID]`
  - お知らせ記事: `https://ai-reboot.io/news/[記事ID]`

## 8. 記事公開のワークフロー

1. **下書き作成**
   - カテゴリーを選択
   - タイトルと概要を入力
   - 本文を作成（HTMLまたはMarkdown）

2. **SEO設定**
   - descriptionを最適化
   - タグを設定
   - サムネイル画像をアップロード

3. **プレビュー確認**
   - 表示崩れがないか確認
   - リンクが正しく動作するか確認

4. **公開**
   - 公開日時を設定
   - 公開ボタンをクリック

## 8. 環境変数の設定

`.env.local`ファイルに以下を設定：

```env
MICROCMS_SERVICE_DOMAIN=your-service-id
MICROCMS_API_KEY=your-api-key
```

## 9. トラブルシューティング

### よくある問題と解決方法

#### マークダウンが表示されない
- `md-content`フィールドに内容が入力されているか確認
- フィールドIDが正確に`md-content`になっているか確認

#### 画像が表示されない
- 画像URLが正しいか確認
- 画像のアクセス権限を確認

#### カテゴリーでフィルタリングされない
- カテゴリーの値が定義された値と一致しているか確認
- 大文字小文字の違いに注意

## 10. ベストプラクティス

### 記事作成時
1. ターゲット読者を明確にする
2. 実践的な内容を含める
3. 視覚的要素（画像、図表）を活用
4. 定期的に更新する

### SEO対策
1. 長尾キーワードを狙う
2. 内部リンクを適切に配置
3. 構造化データを意識した記述
4. モバイルファーストで考える

### パフォーマンス
1. 画像を最適化する
2. 不要なHTMLタグを避ける
3. 外部リンクは新規タブで開く設定
